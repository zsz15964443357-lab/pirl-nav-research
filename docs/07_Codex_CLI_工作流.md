# 07 Codex CLI 工作流

## 基本原则

Codex CLI 只能按阶段执行，不允许一次性读取全部文档后自由发挥。

每次任务必须遵守：

1. 只读取当前 task 文件指定的文档；
2. 只完成当前阶段目标；
3. 不提前实现后续阶段；
4. 产物必须可审查；
5. 生成的可视化、日志、测试结果必须能被网页端复审。

## 推荐执行方式

每次让 Codex CLI 读取一个任务文件：

```text
codex_tasks/TASK_00_仓库初始化.md
codex_tasks/TASK_01_场景生成器.md
codex_tasks/TASK_02_可视化流水线.md
...
```

不要一次让它执行“搭建完整项目”。

## 首次任务边界

首次任务只允许：

- 建立代码目录；
- 加入最小 README；
- 加入空模块或 `.gitkeep`；
- 建立测试目录；
- 不实现 PyBullet；
- 不实现 RL。

## 每阶段输出

每阶段至少输出：

- 变更摘要；
- 如何运行；
- 生成的审查材料；
- 已知限制；
- 下一阶段建议。

## 防止额度浪费

如果 Codex CLI 开始做以下事情，应立即停止：

- 未经允许引入大型框架；
- 跳过场景可视化直接训练；
- 先写复杂网络而不是场景和指标；
- 把 PyBullet / ROS2 作为主创新；
- 修改研究主线文档而未说明理由。
