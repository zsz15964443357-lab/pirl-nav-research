# 01 架构决策

## 平台总体选择

当前确认采用三层平台架构：

```text
辅助原型：自建轻量 2.5D Gymnasium 环境
主训练：PyBullet-based UAV-IntentRisk Env
最终验证：Gazebo / ROS2 Runtime，后期可选 PX4 SITL
```

## 为什么保留自建轻量 Gym

自建轻量环境用于：

- 快速检查 reward / cost / metric 是否合理；
- 快速验证 scenario generator；
- 快速调试 AT、SDI、risk exposure 等指标；
- 作为算法沙盘，而不是最终可信度证据。

## 为什么 PyBullet 做主训练

由于本项目不要求训练速度最快，而是要求结果可信、动作合理、可发表，PyBullet 比纯自建 2.5D 环境更适合作为主训练平台。

PyBullet 应体现：

- 无人机速度、加速度、yaw-rate 约束；
- 碰撞检测与 clearance 计算；
- 遮挡体几何；
- 动态对象运动；
- 观测噪声、延迟、dropout；
- 控制延迟与执行误差。

## 为什么还需要 Gazebo / ROS2

Gazebo / ROS2 不作为训练主平台，而作为最终系统可信度验证层，用于验证：

- 传感器、里程计、目标跟踪、策略推理、安全监督器的节点链路；
- policy_cmd / safe_cmd 是否有延迟和堆积；
- C++ / ONNX 推理是否满足实时链路要求；
- p95 / p99 latency 和 deadline miss。

## 统一接口原则

三个平台应尽量共享同一套高层观测与动作语义。

观测：

```text
ego state
relative goal
local occupancy 或 scan
Top-K object tracks
intent-risk features
last action
```

动作：

```text
body-frame velocity command 或 acceleration command
```

第一版不直接输出电机推力，避免论文重心从导航与风险决策转移到底层飞控。
