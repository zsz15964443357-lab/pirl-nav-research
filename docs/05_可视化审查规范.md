# 05 可视化审查规范

## 硬规则

**未通过可视化审查，不允许进入固定测试集；未通过场景可视化审查，不允许启动大规模训练。**

## 三层可视化

### 1. 静态场景预览

用于审查地图、起点、终点、静态障碍物、动态对象初始位置、遮挡物位置是否合理。

输出：

```text
layout.png
multi_seed_grid.png
```

### 2. 时间展开可视化

用于审查动态对象未来轨迹、启动时刻、遮挡出现、多意图分支是否合理。

输出：

```text
timeline.png
future_trajectory_overlay.png
```

### 3. 策略 rollout 可视化

用于审查训练后的策略行为，包括提前避险、过度保守、shield 依赖和动作抖动。

输出：

```text
rollout.gif
risk_overlay.png
intervention_overlay.png
```

## 统一图例

- UAV start：蓝色圆点；
- Goal：绿色星标；
- Static obstacles：灰色块；
- Occluders：黑色块；
- Dynamic objects：橙色圆点；
- Potential intent-risk objects：红色圆点；
- Nominal UAV corridor：蓝色虚线；
- Future object trajectories：箭头；
- Shield intervention：紫色叉号或标记点。

## 场景审查清单

每个候选 seed 至少检查：

1. 起点到终点是否存在可行路径；
2. 风险对象未来轨迹是否与 nominal corridor 有潜在交集；
3. 场景不是直线无风险，也不是必撞无解；
4. 意图行为是否符合语义和上下文；
5. 遮挡场景中对象是否真的被遮挡；
6. easy / medium / hard 是否有明显难度区分；
7. risk field 不应全图高风险；
8. semantic_distractor 不应错误阻断主路径；
9. near miss、AT、SDI、risk exposure 能正常记录。

## 审查结果

每个 seed 标记为：

```text
PASS：可进入候选测试集
FIX：需要修正配置后重新生成
REJECT：不进入测试集
```

所有 PASS seed 记录到 `experiments/manifests/reviewed_scenarios.csv`。
