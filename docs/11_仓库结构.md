# 11 仓库结构

本文件定义 PIRL-Nav 的仓库结构契约。后续 Codex CLI、人工实现和论文实验都应以此为准。

## 设计原则

1. **文档先于实现**：场景、指标、训练协议和论文贡献边界必须先被写清楚。
2. **小步可审查**：每个阶段只增加一类能力，并产生可视化或可测试产物。
3. **实验可复现**：所有训练、评估、固定测试集都必须能追溯到 manifest、config、commit 和 seed。
4. **路径工具友好**：代码、配置、CI、脚本路径优先使用 ASCII；中文内容可以保留在 Markdown 文档中。
5. **大文件外置**：不提交训练日志、视频、rosbag、模型权重和未筛选图片。

## 模块职责边界

| 模块 | 职责 | 当前阶段允许内容 |
|---|---|---|
| `pirl_nav.sim` | Gymnasium / PyBullet 环境封装 | 只放接口说明，不实现环境 |
| `pirl_nav.perception` | 检测、语义、跟踪输入适配 | 只放数据契约说明 |
| `pirl_nav.intent` | 潜在意图先验与预测器 | 只放模块边界说明 |
| `pirl_nav.risk` | 动作相关未来风险估计 | 只放风险语义说明 |
| `pirl_nav.rl` | 训练算法和策略接口 | Stage 0 不写训练循环 |
| `pirl_nav.safety` | Safety supervisor / shield | Stage 0 不写控制逻辑 |
| `pirl_nav.evaluation` | 指标、聚合、统计报告 | 先依据指标契约写测试 |
| `pirl_nav.visualization` | 场景预览和 rollout 可视化 | Stage 2 开始实现 |

## 文档追溯规则

所有实现必须能追溯到：

- `docs/` 中的研究决策；
- `experiments/scenario_specs/` 中的场景规格；
- `experiments/manifests/` 中的训练 / 验证 / 测试 manifest；
- `experiments/review_checklists/` 中的审查规则；
- `codex_tasks/` 中的当前阶段任务。

## 禁止提交内容

不要提交：

- 大规模训练日志；
- wandb / tensorboard 运行目录；
- rosbag；
- 大型视频；
- 模型权重；
- 未筛选图片；
- `build/`、`dist/`、`*.egg-info/`；
- `.venv/`、`.pytest_cache/`、`.ruff_cache/`。

需要保留的代表性图像后续可放到 `docs/figures/`，但必须经过人工筛选，并在 PR 中说明用途。
