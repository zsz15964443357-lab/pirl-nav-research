# 11 仓库结构

本文件定义 PIRL-Nav 的仓库结构契约。后续 Codex CLI、人工实现和论文实验都应以此为准。

## 设计原则

1. **文档先于实现**：场景、指标、训练协议和论文贡献边界必须先被写清楚。
2. **小步可审查**：每个阶段只增加一类能力，并产生可视化或可测试产物。
3. **一功能一文档**：一个长期功能只保留一个权威文档；重复、过期或草案文件必须合并进权威文档或删除。
4. **实验可复现**：所有训练、评估、固定测试集都必须能追溯到 manifest、config、commit 和 seed。
5. **路径工具友好**：代码、配置、CI、脚本路径优先使用 ASCII；中文内容可以保留在 Markdown 文档中。
6. **大文件外置**：不提交训练日志、视频、rosbag、模型权重和未筛选图片。

## 文档所有权

| 功能 | 权威文档 |
|---|---|
| 项目入口、研究定位、全局禁令 | `README.md` |
| 阶段路线图和 stage gate | `ROADMAP.md` |
| 仓库结构契约 | `docs/11_仓库结构.md` |
| Codex CLI / skill / 文件卫生规则 | `docs/15_codex_cli_academic_research_suite.md` |
| 实验协议、训练顺序、运行模式 | `docs/03_experimental_protocol.md` |
| 指标定义 | `docs/04_metrics_contract.md` |
| 场景 benchmark / family / difficulty | `docs/05_scenario_benchmark.md` |
| 每阶段执行任务 | `codex_tasks/TASK_xx_*.md` |
| 场景规格 | `experiments/scenario_specs/*.yaml` |
| 场景集合状态 | `experiments/manifests/*.yaml` |
| 阶段审查结论 | 对应 stage audit 文档，例如 `docs/18_stage2_visualization_audit_2026-06-30.md` |

不要为同一功能新增 `_v2`、`_final`、`_updated`、`review_pass` 或 `review_fix` 文档。应直接修改对应权威文档并在 PR 描述中记录原因。

## 模块职责边界

| 模块 | 职责 | 当前阶段允许内容 |
|---|---|---|
| `pirl_nav.sim` | Gymnasium / PyBullet 环境封装 | Stage 3 只允许最小 reset / step / render / info gate |
| `pirl_nav.perception` | 检测、语义、跟踪输入适配 | 只放数据契约说明 |
| `pirl_nav.intent` | 潜在意图先验与预测器 | 只放模块边界说明 |
| `pirl_nav.risk` | 动作相关未来风险估计 | 只放风险语义说明 |
| `pirl_nav.rl` | 训练算法和策略接口 | Stage 5 前不写训练循环 |
| `pirl_nav.safety` | Safety supervisor / shield | Stage 6 前不写完整控制逻辑 |
| `pirl_nav.evaluation` | 指标、聚合、统计报告 | 依据指标契约写测试和聚合 |
| `pirl_nav.visualization` | 场景预览和 rollout 可视化 | Stage 2 已实现场景审查预览 |

## 文档追溯规则

所有实现必须能追溯到：

- `docs/` 中的研究决策；
- `experiments/scenario_specs/` 中的场景规格；
- `experiments/manifests/` 中的训练 / 验证 / 测试 manifest；
- `experiments/review_checklists/` 中的审查规则；
- `codex_tasks/` 中的当前阶段任务。

## 禁止提交内容

不要提交：

- 大规模训练日志；
- wandb / tensorboard 运行目录；
- rosbag；
- 大型视频；
- 模型权重；
- 未筛选图片；
- `build/`、`dist/`、`*.egg-info/`；
- `.venv/`、`.pytest_cache/`、`.ruff_cache/`。

需要保留的代表性图像后续可放到 `docs/figures/`，但必须经过人工筛选，并在 PR 中说明用途。
