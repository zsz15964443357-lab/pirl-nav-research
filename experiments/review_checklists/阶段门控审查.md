# 阶段门控审查清单

每个阶段结束后必须给出 PASS / FIX / REJECT。

## 通用检查

- [ ] 当前阶段只完成了对应 task 文件要求。
- [ ] 没有提前实现后续阶段。
- [ ] 输出产物可复现。
- [ ] 有清楚的运行说明。
- [ ] 有可视化、日志或测试证据。
- [ ] 已记录已知问题和下一步。

## 研究主线检查

- [ ] 没有偏离预测式意图风险主线。
- [ ] 没有把平台工程包装成主创新。
- [ ] 没有只优化 success rate。
- [ ] 没有隐藏失败案例。

## 结论

- [ ] PASS：进入下一阶段。
- [ ] FIX：当前阶段修正后重审。
- [ ] REJECT：回到方案设计。
